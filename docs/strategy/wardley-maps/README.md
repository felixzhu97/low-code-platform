# 沃德利地图文档

本目录包含 Felix 低代码平台的沃德利地图（Wardley Map），这是一种战略地图，用于可视化价值网络和组件演化阶段，帮助理解平台各组件在演化图谱中的位置，为技术决策和战略规划提供重要参考。

## 目录说明

沃德利地图是一种战略地图工具，由 Simon Wardley 创建，用于：

- 可视化价值网络（从用户需求到基础设施的完整链条）
- 展示组件演化阶段（从创新到商品化的过程）
- 指导技术决策和战略规划
- 识别竞争优势和投资重点

## 文件清单

### 沃德利地图

- **文件**：`wardley-map.puml`
- **描述**：展示平台的价值网络和组件演化阶段
- **核心内容**：

#### 价值网络（Y 轴 - 可见性）

从用户需求层到基础设施层的完整价值链条：

- **用户需求层**：页面设计师、开发者、终端用户的需求
- **应用层**：可视化编辑器、模板系统、协作功能
- **平台层**：组件系统、数据绑定、主题系统、代码生成引擎
- **基础设施层**：部署服务、存储服务、CDN 服务、数据库服务、认证服务

#### 演化阶段（X 轴 - 演化）

各组件的演化位置：

- **Genesis（创新）**：AI 辅助设计、智能布局推荐等新兴技术

  - 高不确定性，需要持续创新和探索
  - 具有高差异化和竞争优势

- **Custom Built（定制构建）**：可视化编辑器、实时协作等平台核心差异化能力

  - 平台的核心竞争力
  - 需要持续投入和优化

- **Product（产品化）**：组件系统、模板系统、代码生成等标准化产品

  - 已经产品化，可以稳定提供
  - 具有标准化特征

- **Commodity（商品化）**：CDN、存储、数据库、认证等标准化云服务
  - 标准化的基础设施服务
  - 可以轻松替换和采购

#### 依赖关系

展示组件之间的依赖和价值流动，帮助理解：

- 哪些组件是核心差异化能力
- 哪些组件可以外包或采购
- 投资重点应该放在哪里

## 使用方法

### 查看图表

这些 PlantUML 文件可以使用以下工具渲染：

- **在线工具**：[PlantUML Online](http://www.plantuml.com/plantuml/)
- **VS Code 插件**：PlantUML
- **命令行工具**：PlantUML CLI

### 命令行生成图片

```bash
# 使用 PlantUML CLI 生成图片
plantuml -tpng docs/wardley-maps/wardley-map.puml
```

## 使用场景

沃德利地图可以用于：

- **战略规划**：识别核心竞争力和投资重点
- **技术决策**：决定哪些组件应该自建，哪些可以采购
- **资源分配**：将资源集中在高价值的差异化能力上
- **风险评估**：识别对商品化组件的过度依赖
- **创新规划**：识别需要创新的领域

## 关键洞察

基于沃德利地图的分析：

1. **核心差异化能力**（Custom Built）

   - 可视化编辑器：平台的核心价值
   - 实时协作：独特的协作体验

2. **产品化能力**（Product）

   - 组件系统、模板系统、代码生成：标准化产品，提供稳定价值

3. **基础设施**（Commodity）

   - CDN、存储、数据库、认证：可以采购标准化服务

4. **创新方向**（Genesis）
   - AI 辅助设计：未来的竞争优势
   - 智能布局：提升用户体验
